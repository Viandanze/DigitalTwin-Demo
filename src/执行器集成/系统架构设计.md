# 执行器集成实战项目系统架构设计

## 项目概述

本项目基于Week 2前三天（Day 8树莓派5基础、Day 9多传感器数据采集、Day 10电机控制理论）的学习成果，设计并实现一个数字孪生系统中的闭环控制演示系统。由于硬件可能仍在采购中，本次实战以模拟和代码实现为主，为实际硬件到手后的调试做准备。

## 系统架构

### 硬件架构

采用分层控制架构，树莓派5作为上位机，Arduino UNO作为下位机，通过串口/UART进行双向数据交换：

```
┌─────────────────────────────────────────────────────────────┐
│                     上位机：树莓派5                          │
├─────────────────────────────────────────────────────────────┤
│ 功能：                                                  │
│  • 逻辑决策（基于模拟传感器数据）                        │
│  • 控制指令生成                                          │
│  • 数字孪生可视化（Three.js）                           │
│  • 串口通信（与Arduino）                                │
│  • 数据记录与分析                                       │
└──────────────────────────────┬──────────────────────────────┘
                               │ UART串口 (GPIO14/TXD, GPIO15/RXD)
                               ▼
┌─────────────────────────────────────────────────────────────┐
│                     下位机：Arduino UNO                      │
├─────────────────────────────────────────────────────────────┤
│ 功能：                                                  │
│  • 实时电机控制                                          │
│  • 执行器状态监控                                        │
│  • 串口指令解析                                          │
│  • 状态反馈上传                                          │
└───────────────────────┬─────────────────────────────────────┘
                        │
         ┌──────────────┼──────────────┐
         ▼              ▼              ▼
    ┌─────────┐   ┌─────────┐   ┌─────────┐
    │直流电机 │   │SG90舵机 │   │步进电机 │
    │(L298N)  │   │         │   │(ULN2003)│
    └─────────┘   └─────────┘   └─────────┘
```

### 硬件接口分配

#### 树莓派5 GPIO分配（BCM编号）

| GPIO | 功能 | 连接目标 | 备注 |
|------|------|----------|------|
| GPIO14 (TXD) | UART发送 | Arduino RX (Pin 0) | 串口通信 |
| GPIO15 (RXD) | UART接收 | Arduino TX (Pin 1) | 串口通信 |
| GPIO23 | 电机方向控制1 | L298N IN1 | 直流电机控制 |
| GPIO24 | 电机方向控制2 | L298N IN2 | 直流电机控制 |
| GPIO18 | PWM调速信号 | L298N ENA | 软件PWM（硬件PWM优先） |

#### Arduino UNO引脚分配

| 引脚 | 功能 | 连接目标 | 备注 |
|------|------|----------|------|
| D9 | PWM舵机控制 | SG90信号线 | 支持0-180度角度控制 |
| D8 | 步进电机控制1 | ULN2003 IN1 | 28BYJ-48相位A |
| D7 | 步进电机控制2 | ULN2003 IN2 | 28BYJ-48相位B |
| D6 | 步进电机控制3 | ULN2003 IN3 | 28BYJ-48相位C |
| D5 | 步进电机控制4 | ULN2003 IN4 | 28BYJ-48相位D |

### 数据流设计

#### 上行数据流（传感器数据 → 树莓派）

```
模拟传感器数据生成 → 数据处理与融合 → MQTT发布 → Three.js可视化
      (Python)           (Python)        (模拟)        (Web前端)
```

**模拟传感器数据源：**
1. 温度传感器：随机生成15-35℃范围，模拟温室环境
2. 超声波测距：随机生成5-45cm水位，模拟水箱液位
3. 光照强度：随机生成0-100%范围，模拟昼夜变化

#### 下行数据流（控制指令 → 执行器）

```
Three.js控制面板 → 控制逻辑决策 → 串口指令编码 → Arduino解析执行
   (Web前端)         (Python)         (Python)         (Arduino)
```

**控制指令格式：**
```
MOTOR:DIR:SPEED     # 直流电机控制，DIR=0/1(正/反)，SPEED=0-100
SERVO:ANGLE         # 舵机角度控制，ANGLE=0-180
STEPPER:DIR:STEPS   # 步进电机控制，DIR=0/1，STEPS=步数
```

### 控制流设计

#### 闭环控制流程

```
1. 传感器数据采集（模拟）
   ↓
2. 状态评估与决策（基于预设规则）
   ↓
3. 控制指令生成（Python逻辑）
   ↓
4. 串口指令下发（树莓派 → Arduino）
   ↓
5. 执行器动作（Arduino实时控制）
   ↓
6. 状态反馈采集（Arduino监控）
   ↓
7. 反馈数据上传（Arduino → 树莓派）
   ↓
8. 可视化更新（Three.js界面）
   ↓
   └── 返回步骤1，形成闭环
```

#### 控制决策规则

**场景1：基于超声波测距的自动避障小车（模拟）**
- 规则：当距离 < 20cm时，停止前进并右转避障
- 控制指令：`MOTOR:0:0`（停止） → `MOTOR:1:50`（右转2秒） → `MOTOR:0:70`（继续前进）

**场景2：基于温湿度的智能温室环境控制（模拟）**
- 温度规则：当温度 > 28℃时，启动通风扇；温度 < 18℃时，启动加热器
- 湿度规则：当湿度 < 40%时，启动加湿器；湿度 > 80%时，启动除湿器
- 控制指令：`MOTOR:0:60`（通风扇）或 `SERVO:45`（遮阳帘角度）

### 软件架构

#### 分层设计

```
┌─────────────────────────────────────────────┐
│            应用层（Presentation Layer）     │
├─────────────────────────────────────────────┤
│ • Three.js可视化界面（HTML/CSS/JavaScript） │
│ • 控制面板（滑块、按钮、状态显示）          │
│ • 实时数据图表                            │
└─────────────────────────────────────────────┘
┌─────────────────────────────────────────────┐
│            逻辑层（Logic Layer）            │
├─────────────────────────────────────────────┤
│ • Python主控制程序（control_main.py）      │
│ • 传感器数据模拟与融合                     │
│ • 控制决策引擎（基于规则/模糊逻辑）         │
│ • 串口通信管理（serial_manager.py）        │
└─────────────────────────────────────────────┘
┌─────────────────────────────────────────────┐
│            驱动层（Driver Layer）           │
├─────────────────────────────────────────────┤
│ • Arduino固件（actuator_controller.ino）   │
│ • 电机驱动库（L298N、舵机、步进电机）       │
│ • 状态反馈采集                            │
└─────────────────────────────────────────────┘
```

#### 关键模块

1. **传感器模拟模块** (`sensor_simulator.py`)
   - 生成模拟传感器数据
   - 添加可控噪声和异常
   - 支持手动干预测试

2. **控制决策模块** (`control_engine.py`)
   - 基于规则的控制逻辑
   - 支持多场景切换
   - 决策日志记录

3. **串口通信模块** (`serial_manager.py`)
   - 指令编码/解码
   - 超时重传机制
   - 通信状态监控

4. **Arduino固件** (`actuator_controller.ino`)
   - 指令解析与执行
   - 多执行器协同控制
   - 状态反馈上报

### 通信协议

#### 串口协议规范

- **波特率**：115200 bps
- **数据位**：8位
- **停止位**：1位
- **校验位**：无

#### 指令格式

**树莓派 → Arduino（控制指令）**
```
<CMD>:<PARAM1>:<PARAM2>\n
```

**Arduino → 树莓派（状态反馈）**
```
<DEVICE>:<STATUS>:<VALUE>\n
```

#### 具体指令集

| 指令 | 格式 | 说明 | 示例 |
|------|------|------|------|
| 直流电机控制 | `MOTOR:DIR:SPEED` | DIR=0/1(正/反)，SPEED=0-100 | `MOTOR:0:75` |
| 舵机角度控制 | `SERVO:ANGLE` | ANGLE=0-180 | `SERVO:90` |
| 步进电机控制 | `STEPPER:DIR:STEPS` | DIR=0/1，STEPS=步数 | `STEPPER:1:200` |
| 状态查询 | `STATUS:ALL` | 查询所有设备状态 | `STATUS:ALL` |

#### 反馈格式

| 设备 | 格式 | 说明 | 示例 |
|------|------|------|------|
| 直流电机 | `MOTOR:SPEED:<VALUE>` | 当前PWM占空比 | `MOTOR:SPEED:75` |
| 舵机 | `SERVO:ANGLE:<VALUE>` | 当前角度 | `SERVO:ANGLE:90` |
| 步进电机 | `STEPPER:POS:<VALUE>` | 当前位置（步） | `STEPPER:POS:1024` |
